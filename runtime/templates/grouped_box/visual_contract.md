# 视觉合同

## 文字来源

- 每个原始数据列必须使用 `Category | Group` 表头。横轴分类逐字来自 `Category`，图例逐字来自
  `Group`；渲染器不得把它们替换成 `Group 1`、`N1` 或其他占位文字，也不得静默截断。
- X/Y 轴标题默认分别为 `Condition` 与 `Observed value`。用户确认的精确标题必须通过冻结
  RenderPlan 的 `axis_title_overrides` 传入，并同时用于 Python 规则预览、Origin 成图和对象反读；
  不修改原始 CSV 表头或数据。
- 箱体与表头按一一对应关系绘制；缺失尾部允许造成不同样本量，但不得补值或补列。

## 样本量标签

- 每个箱体显示一个小写 `n=<有效非空观测数>`，例如 `n=14`。禁止写成孤立的 `N1`、只显示
  首位数字或复用其他箱体的样本量。
- `n=` 标签必须附着到数据坐标（Origin `attach=2`），位于绘图区内部、箱体下方的专用留白带，
  不得漂到横轴分类、X 轴标题或页面外边距中。
- Y 轴范围需要为样本量带预留空间；预览与 Origin 使用同一冻结范围和同一相对位置。

## 视觉层级与字体

- 组别使用低饱和紫蓝/珊瑚同族配色。半透明箱体、深色原始点和粗中位线共同表达分布；
  不使用彩虹配色，不自动计算或添加 p 值、显著性括号或星号。
- 图例固定在绘图区上方中央空白区，不显示外框；色块与文字均为可编辑 Origin 对象并附着到
  数据坐标，不覆盖箱体或原始点。
- 页面按分类数、组数和标签长度自适应。轴标题 20 pt、刻度 17 pt、图例 16 pt、样本量标签
  12.48 pt。所有轴标题、刻度、图例、分类文字和 `n=` 标签统一使用 Arial；Origin API 的
  `fsize`/`label.pt` 直接使用 pt，不做 `×500` 换算（`×500` 只适用于曲线 `set -w`）。

## 必须反读

- 分类文字与 `category_order` 完全一致，组别文字与 `group_order` 完全一致。
- 每个样本量对象的文本、计数、X/Y 数据坐标、`attach=2`、字号和 Arial font code。
- X/Y 轴标题文本与字体、X/Y 刻度字体、无原生带框图例、无截断或占位标签。
- 页面/图层、箱体、原始点和源数据未修改状态；成功仍必须包含可编辑 OPJU、PNG、PDF、TIF
  以及人工视觉检查，不能只凭 PNG 汇报。
